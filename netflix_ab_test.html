Drive Clicks?

A Netflix-style A/B experiment simulating 1,000 users, two competing artwork versions, and a statistical test to find out whether the winner is real — or just lucky.

The Business Problem

Why Netflix Tests Everything

Netflix operates in one of the most competitive attention markets in history. Every second a subscriber spends scrolling without clicking something is a second closer to them closing the app. The single biggest lever Netflix has found for driving that first click — and ultimately a play-start — is artwork: the thumbnail image displayed for every title on your home screen.

The same movie can be shown with a close-up of a lead actor's face, an action scene, a mood-driven landscape, or a title card. Netflix doesn't guess which one works best. It runs a controlled experiment — an A/B test — showing different versions to randomly selected groups of real members and measuring which version produces more clicks. The winning artwork is then shown to everyone.

This project replicates that experimental framework in Python, from the random assignment of users all the way through to a formal statistical test — the same logic Netflix's internal experimentation platform uses before any product change gets shipped to hundreds of millions of subscribers.

🎨

The Problem

Which artwork version makes more subscribers click on a title — and is the difference real or random chance?

🔬

The Method

Randomly assign users to two groups. Measure clicks. Run a statistical test to decide if the winner is trustworthy.

📈

The Outcome

A data-driven recommendation: ship the better artwork — but only if the math says the difference is real.

Concepts

A/B Testing & Machine Learning

An A/B test is one of the simplest and most powerful tools in data-driven business. You take two versions of something — a webpage, a product feature, an email subject line, an artwork thumbnail — and randomly show each version to a different group of users. Then you measure which version produces better outcomes.

The word "randomly" is doing a lot of work in that sentence. Random assignment is what makes the comparison fair: because users are randomly split, the two groups should be statistically identical in every way that matters (age, taste, viewing habits) except for the version of the artwork they saw. Any difference in click rates can therefore be attributed to the artwork itself, not to the kind of people in each group.

In the broader world of machine learning and AI, A/B testing plays a critical role as the final validation step. A recommendation algorithm, a ranking model, or a personalization system might look great in offline evaluation, but the only way to know whether it actually improves real user behavior is to run a live experiment. Netflix's ML models — the ones that decide which titles to surface for each subscriber — are continuously validated this way.

What I Built

Three-Step Experiment

1

Simulation

1,000 users were created and randomly assigned to one of two artwork versions — A or B — with an even 50/50 split. Artwork A has a true click probability of 24%; Artwork B has a true click probability of 30%. Each user's click was then simulated using a Bernoulli trial — essentially a weighted coin flip — and stored in a pandas DataFrame alongside their user ID and group assignment.

2

Measurement

For each group, the code calculated the total number of users, total number of clicks, and the click-through rate (CTR) — the percentage of users who clicked. Results were displayed in a clean summary table and a bar chart that makes the difference between the two versions immediately visible.

3

Statistical Testing

A two-proportion z-test was run to determine whether the observed difference in CTR between the two groups was statistically significant — meaning unlikely to have occurred by random chance. The test produced a p-value, a z-statistic, and a 95% confidence interval on the lift, giving a rigorous basis for the final recommendation.

✅ Statistically Significant

The two-proportion z-test confirmed that the difference between Artwork A and Artwork B is statistically significant at the α = 0.05 level. The p-value fell well below the 5% threshold, and the 95% confidence interval on the lift did not include zero — meaning the observed advantage of Artwork B is almost certainly real, not a product of random noise.

The z-statistic was large enough to rule out random chance as the explanation for the difference.
The confidence interval places the true lift between approximately +2 and +10 percentage points.
At Netflix's scale, a +6 pp improvement in click-through rate translates to millions of additional play-starts per week.
Random assignment ensures the result is attributable to the artwork itself, not to any difference in the user groups.
The Work

Complete Python Code

The code below is the complete, runnable implementation — all three steps in a single block. It was written for Google Colab and requires no additional installation beyond the standard scientific Python stack (NumPy, Pandas, Matplotlib, SciPy).

netflix_ab_test.py · Python 3 
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import matplotlib.ticker as mtick
from scipy import stats

# ── 1. Reproducibility ────────────────────────────────────────────────────
np.random.seed(42)

# ── 2. Experiment parameters ──────────────────────────────────────────────
N_USERS  = 1_000
PROB_A   = 0.24   # click probability for Artwork A (control)
PROB_B   = 0.30   # click probability for Artwork B (treatment)

# ── 3. User IDs ───────────────────────────────────────────────────────────
user_ids = np.arange(1, N_USERS + 1)

# ── 4. Random assignment (50 / 50 split) ──────────────────────────────────
assignments = np.random.choice(["A", "B"], size=N_USERS, p=[0.5, 0.5])

# ── 5. Simulate clicks ────────────────────────────────────────────────────
click_probs = np.where(assignments == "A", PROB_A, PROB_B)
clicks      = np.random.binomial(n=1, p=click_probs)

# ── 6. Assemble DataFrame ─────────────────────────────────────────────────
df = pd.DataFrame({
    "user_id" : user_ids,
    "artwork" : assignments,
    "clicked" : clicks
})

# ── 7. Summary metrics ────────────────────────────────────────────────────
summary = (
    df.groupby("artwork")["clicked"]
    .agg(
        total_users  = "count",
        total_clicks = "sum",
        ctr          = "mean"
    )
    .reset_index()
)
summary["ctr_pct"] = (summary["ctr"] * 100).round(2)

print("=" * 40)
print("        EXPERIMENT SUMMARY")
print("=" * 40)
print(f"{'Artwork':<10} {'Users':>8} {'Clicks':>8} {'CTR':>8}")
print("-" * 40)
for _, row in summary.iterrows():
    print(f"{row['artwork']:<10} {int(row['total_users']):>8} "
          f"{int(row['total_clicks']):>8} {row['ctr_pct']:>7.2f}%")
print("=" * 40)

lift = summary.loc[summary["artwork"] == "B", "ctr"].values[0] \
     - summary.loc[summary["artwork"] == "A", "ctr"].values[0]
print(f"\nAbsolute lift (B − A): {lift*100:.2f} percentage points")

# ── 8. Bar chart ──────────────────────────────────────────────────────────
fig, ax = plt.subplots(figsize=(7, 5))
colors  = ["#564d9b", "#e50914"]
bars    = ax.bar(summary["artwork"], summary["ctr"],
               color=colors, width=0.45, zorder=3)

for bar, val in zip(bars, summary["ctr_pct"]):
    ax.text(bar.get_x() + bar.get_width() / 2,
            bar.get_height() + 0.003,
            f"{val:.2f}%", ha="center", va="bottom",
            fontsize=13, fontweight="bold")

y_a = summary.loc[summary["artwork"] == "A", "ctr"].values[0]
y_b = summary.loc[summary["artwork"] == "B", "ctr"].values[0]
ax.annotate("",
    xy=(1, y_b - 0.005), xytext=(1, y_a + 0.005),
    arrowprops=dict(arrowstyle="<->", color="black", lw=1.6))
ax.text(1.24, (y_a + y_b) / 2, f"+{lift*100:.2f} pp lift",
        va="center", fontsize=10)

ax.set_title("Click-Through Rate by Artwork Version",
             fontsize=14, fontweight="bold", pad=14)
ax.set_xlabel("Artwork Version", fontsize=12)
ax.set_ylabel("Click-Through Rate", fontsize=12)
ax.yaxis.set_major_formatter(mtick.PercentFormatter(xmax=1, decimals=0))
ax.set_ylim(0, max(summary["ctr"]) * 1.25)
ax.yaxis.grid(True, linestyle="--", alpha=0.6, zorder=0)
ax.spines[["top", "right"]].set_visible(False)
plt.tight_layout()
plt.savefig("ctr_comparison.png", dpi=150, bbox_inches="tight")
plt.show()

# ── 9. Statistical test (two-proportion z-test) ───────────────────────────
clicks_a = int(summary.loc[summary["artwork"] == "A", "total_clicks"].values[0])
clicks_b = int(summary.loc[summary["artwork"] == "B", "total_clicks"].values[0])
n_a      = int(summary.loc[summary["artwork"] == "A", "total_users"].values[0])
n_b      = int(summary.loc[summary["artwork"] == "B", "total_users"].values[0])
ctr_a    = clicks_a / n_a
ctr_b    = clicks_b / n_b

p_pool   = (clicks_a + clicks_b) / (n_a + n_b)
se_pool  = np.sqrt(p_pool * (1 - p_pool) * (1/n_a + 1/n_b))
z_stat   = (ctr_b - ctr_a) / se_pool
p_value  = stats.norm.sf(abs(z_stat)) * 2

ci_low   = (ctr_b - ctr_a) - 1.96 * se_pool
ci_high  = (ctr_b - ctr_a) + 1.96 * se_pool
alpha    = 0.05

print("\n" + "=" * 40)
print("      STATISTICAL TEST RESULTS")
print("=" * 40)
print(f"  CTR  —  Artwork A : {ctr_a*100:.2f}%")
print(f"  CTR  —  Artwork B : {ctr_b*100:.2f}%")
print(f"  Absolute lift     : {lift*100:.2f} pp")
print(f"  Z-statistic       : {z_stat:.4f}")
print(f"  P-value           : {p_value:.4f}")
print(f"  95% CI on lift    : [{ci_low*100:.2f} pp,  {ci_high*100:.2f} pp]")
print("-" * 40)

if p_value < alpha:
    verdict = "SIGNIFICANT ✅  — likely a real difference"
else:
    verdict = "NOT SIGNIFICANT ❌ — could be due to chance"
print(f"  Result (α = 0.05) : {verdict}")
print("=" * 40)
Business Recommendation

Ship Artwork B.

The data is clear and the statistics back it up. Artwork B outperformed Artwork A by 6 percentage points in click-through rate, and a formal statistical test confirmed that this gap is almost certainly real — not a product of random variation in our sample.

For a company the size of Netflix, with over 300 million subscribers globally, even a 1-percentage-point improvement in thumbnail click rates compounds into an enormous number of additional play-starts every day. A 6-point improvement is not incremental — it's significant.

  • Replace Artwork A with Artwork B in the production content catalog immediately.
  • Use the confidence interval (+2 pp to +10 pp) to size the expected business impact before the rollout.
  • Monitor post-launch CTR for one to two weeks to confirm the experiment result holds in production.
  • Apply the same experimental framework to other titles — artwork testing should be a continuous, systematic practice, not a one-time event.
  • Explore whether the lift holds across different subscriber segments (mobile vs. TV, new vs. returning members) to determine if personalized artwork could drive even higher engagement.
